# 개발 일지: v0.2.0 로그인 검증과 QA 보강

**일자:** 2026-03-26
**관련 버전:** v0.2.0
**관련 역할:** build, qa, ship

## 배경 (Context)
PRD는 비로그인 사용자의 접근 차단과 레퍼런스 저장 흐름을 MVP 핵심 요구사항으로 정의했습니다. 이번 사이클에서는 프론트엔드 셸 상태에서 멈춰 있던 로그인 보호 흐름과 레퍼런스 등록 흐름을 실제 동작 수준으로 연결하고, 릴리즈 직전 QA까지 통과하는 것이 필요했습니다.

## 문제 (Problem)
초기 구현만으로는 두 가지 문제가 남아 있었습니다.

1. 로그인 화면과 대시보드가 시각적으로만 존재하고 인증 상태에 따른 라우팅 보호가 없었습니다.
2. 자동화 테스트는 통과했지만, 브라우저 감사에서는 로그인 버튼 색상 대비 부족, 한글 `word-break` 미적용, 루트 한글 폰트 스택 부재, 링크 터치 타겟 부족이 드러났습니다.

## 시도한 것들 (Attempts)
1. **기능 흐름 연결:** 로그인 폼 검증, localStorage 기반 인증 상태 저장, 대시보드 보호 라우팅, 레퍼런스 등록 모달 제출을 먼저 연결했습니다.
2. **테스트 우선 검증:** Go 핸들러 테스트와 Vitest 기반 프론트엔드 테스트를 추가해 기능 회귀선을 먼저 만들었습니다.
3. **브라우저 감사 적용:** `miluju_full_audit`, `miluju_a11y_audit`, `miluju_korean_audit`, `miluju_design_token_audit`를 로그인 화면에 실행해 시각 QA가 아니라 규칙 기반 결함을 잡았습니다.
4. **QA 후 수정:** 버튼 primary 색상을 더 짙게 조정하고, 루트 CSS 토큰과 한글 폰트 스택을 추가했으며, 로그인 라벨/보조 링크에 `text-body-ko`와 터치 타겟 보강 클래스를 적용했습니다.

## 최종 해결 (Resolution)
최종적으로 로그인 검증과 라우트 보호, 레퍼런스 등록 저장 흐름, 서버 입력값 검증을 구현했습니다. 이후 자동화 테스트와 브라우저 감사를 다시 실행해 접근성 0건, 한글 렌더링 0건, 디자인 토큰 위반 0건을 확인한 뒤 `v0.2.0`으로 릴리즈했습니다.

핵심 변경 축은 다음과 같습니다.
- 프론트엔드 인증/라우팅: 로그인 성공 시 세션을 저장하고 인증되지 않은 사용자를 `/login`으로 리다이렉트
- 레퍼런스 등록: 모달 입력 검증 후 대시보드 목록 상태에 즉시 반영
- 서버 검증: URL, 제목 trim 및 필수값 체크
- QA 보강: WCAG AA 대비, 한글 줄바꿈, 폰트 스택, 터치 타겟 기준 충족

## 배운 것 (Lessons Learned)
- 단위 테스트만 통과했다고 릴리즈 가능한 상태는 아닙니다. 브라우저 기반 감사가 한글 UI와 접근성 결함을 별도로 드러냈습니다.
- 접근성 수정은 컴포넌트 한 곳이 아니라 색상 토큰, 전역 폰트, 텍스트 유틸리티처럼 시스템 레벨에서 같이 잡아야 재발을 줄일 수 있습니다.
- 문서 관점에서는 `.env.example` 부재와 하드코딩된 DB 설정처럼 코드가 허용하는 임시 구조를 README와 STATUS에 명확히 남겨야 다음 작업자가 오해하지 않습니다.
