# [PRD] Refentra v1 (MVP)

**Status:** Approved
**Date:** 2026-03-25
**Branch:** main

## 1. 개요 (Overview)
- **배경 및 목적:** 파편화된 링크와 문서 공유(카톡, 디스코드, 드라이브 등)의 비효율을 해결하기 위해, 팀의 레퍼런스가 한곳에 모이고 빠르게 탐색되는 협업형 지식 통합 아카이브 시스템.
- **타겟 워크플로우:** 팀원이 중요한 URL 레퍼런스를 업로드 창에 입력하고 설명을 추가해 저장하면, 모든 팀원이 리스트에서 해당 링크를 검색/태그 필터링 조건으로 쉽게 찾아볼 수 있음.

## 2. 핵심 요구사항 (Core Requirements)
- [ ] **인증 접근:** 사용자는 유효한 로그인 과정을 거쳐야만 서비스 목록에 접근 가능 (단일 폐쇄형 워크스페이스 구조).
- [ ] **레퍼런스 저장:** 사용자는 URL을 등록할 수 있으며 제목, 부연 설명, 카테고리/태그 정보를 기입할 수 있음.
- [ ] **목록 페이징:** 수백~수천 건의 자료를 탐색할 수 있도록 일정 개수씩 끊어서 탐색 가능한 고전적 페이지네이션 방식 (Page Number) 목록을 제공.
- [ ] **메타데이터 표시:** 업로드 내역에는 '저장일시'와 '업로더 정보'가 명시되어야 함.
- [ ] **검색 및 필터링:** 키워드와 특정 태그를 조합해 저장된 레퍼런스를 빠르게 필터링할 수 있어야 함.

## 3. 데이터 및 상태 정의 (Data & State Models)
- **레퍼런스(Reference) 모델 핵심 구조:**
  - `id`: UUID 식별자
  - `url`: 저장된 링크 주소
  - `title`: URL과 관련된 제목 또는 직접 기입한 제목
  - `description`: 상세 부연 설명 메모
  - `tags`: `[]string` (PostgreSQL JSONB 등 활용)
  - `uploader_id`: 업로더 유저 식별자
  - `created_at`: 저장 시간
- **엔드포인트 (기능 예상 API):**
  - `GET /api/v1/references` (쿼리: `page`, `limit`, `search`, `tags`)
  - `POST /api/v1/references` (URL 및 메타데이터 저장)
  - `GET /api/v1/auth/me` (현재 접근 유저 세션 관리)

## 4. 예외 처리 정책 (Exception Handling)
| 상황 | 대응 방안 | 우선순위 |
|------|---------|---------|
| 비로그인 사용자의 URL/목록 접근 시 | 즉시 `401 Unauthorized` 처리 후 로그인 페이지로 강제 리다이렉트 처리 | P1 |
| DB 타임아웃 및 저장 실패 시 | 서버는 500 계열의 정형화된 JSON 에러를 내려주고, 프론트는 저장 실패 Toast 메시지로 표시 | P1 |
| 짧은 시간에 '저장' 버튼 다중 클릭 (Race) | 저장 로딩 중 버튼 비활성화, 백엔드 중복 검사 로직 보완 (동일 유저-동일 링크 연속 처리 방어) | P2 |

## 5. 기술 스택 (Technology Stack)
- **선택한 접근법:** 모던 SPA 프론트엔드 + Go API 통합 서빙 (단일 바이너리 배포)
- **스택 상세:** 
  - **프론트엔드:** React (Vite 기반) — 화려하고 반응성이 뛰어난 검색/목록 UI 구축에 최적
  - **백엔드:** Go (Echo 또는 Gin 프레임워크) — 빠른 JSON 반환, `go:embed` 기능으로 React 빌드 파일 내장
  - **데이터베이스:** PostgreSQL (명확한 관계형 데이터, JSONB를 이용한 강력한 태그 필터링 확장성)
